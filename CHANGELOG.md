# 更新日志

本项目遵循语义化版本（SemVer）。

## [Unreleased] - dev

> 当前条目对应 `dev` 分支测试内容；合并到 `main` 并正式发布时，再整理为具体版本号。

### 修复
- 修复收藏夹详情弹窗筛选按钮过宽时出现横向滚动条的问题。
- 修复队列看板只显示裸耗时、容易被误解为剩余时间的问题，现在会显示“等待 / 运行 / 上传”含义。
- 修复旧命名文件重命名后未同步本地远端文件记录，可能导致后续 AList 对账误判缺失的问题。
- 修复同一 BV 同时存在于多个收藏夹时，共享下载目录被提前清理导致后续收藏夹目录漏传的问题。
- 修复上传没有产出远端文件记录时仍可能被误标记为已上传/已验证的问题。
- 修复 AList 目录读取失败被当成空目录，进而误判远端文件缺失的问题。
- 修复已有旧文件名模板缺少 `<bvid>` 时仍可能继续产生同名冲突的问题。
- 修复命令行登录仍使用 Web 扫码，与 Web 面板 TV 登录不一致的问题。
- 修复队列 active 状态在重启或内存队列丢失后无法重新入队，导致视频长期卡在未上传的问题。
- 修复全量扫描过程中提前标记完成，导致收藏夹详情误提示“继续扫描后补齐”的问题。

### 优化
- “一键重命名网盘文件”改为“检查旧命名文件”，新增预览、勾选、二次确认、执行结果展示与跳过原因说明。
- 新增 AList 旧命名文件递归扫描与完整路径重命名接口，只允许在当前目标路径下同目录改名，并校验目标冲突。
- 下载任务新增低速 watchdog：运行超过 30 分钟且最近 10 分钟平均增长低于 10KB/s 时自动终止并进入现有重试流程。
- 全局设置补充生效范围说明，明确轮询/并发立即生效，画质、命名模板、AList 路径等只影响新任务。
- 账号与同步区域新增“立即同步 / 状态对账 / 全量扫描并对账”帮助弹窗，支持简要和详细介绍。
- 全局设置标题新增可视化执行流程弹窗，按当前表单值展示轮询、下载、上传、重试与 AList 对账行为。
- 下载结果会在所有目标上传完成后再统一清理，支持同一视频复用到多个收藏夹目录。
- 全量扫描完成后会把本轮未再出现的收藏夹关系标记为非当前收藏，改善取消收藏后的状态展示。
- 默认文件名模板改为 `<videoTitle>-<bvid>`，并对旧配置做加载时迁移。
- BBDown 画质与音频优先级映射调整：Hi-Res / 杜比走 `--encoding-priority`，清晰度走 `--dfn-priority`。
- 任务队列提高并发数后会立即补满可用并发槽。
- 画质重调下载阶段接入原有下载队列，上传替换阶段接入原有上传队列，分别受当前下载/上传并发控制。
- 下载、上传和画质重调任务等待自动重试时继续显示在队列看板中，并展示等待重试倒计时。
- 队列看板纳入画质重调任务与调度状态，按原有“待下载 / 下载中 / 待上传 / 上传中”展示重调下载、上传替换、删除清理和同步/对账排队详情。
- B 站风控/登录异常识别覆盖更多 HTTP 状态码、API code 与中文错误文本。
- 收藏夹索引提示改为区分“扫描未完成”和“B 站报告总数但接口未返回明细”，避免全量完成后继续误导用户补扫。
- 账号卡片新增授权健康状态，明确展示自动刷新是否可用、最近刷新成功时间、失败原因和是否需要重新扫码。
- 旧命名文件重命名改为前端分批提交，避免大量项目一次提交触发请求体过大。
- 画质重调任务改为前端每批 50 项提交，避免大量勾选后触发后端单批安全限制。
- 画质重调会记录新上传文件的目标画质设置，后续检查可升级画质时会跳过已符合当前设置的远端记录；旧记录仅在文件名模板可可靠判断时跳过。
- 新增“检查可升级画质”入口，可按当前画质/编码/Hi-Res/杜比设置预览、勾选并后台重调画质；新版文件会先上传到临时目录并验证，再删除旧远端文件并移动到正式目录。
- AList 对账参数上限提高：并发和限速最高可设为 100，每轮最多补传数量最高可设为 1000。
- 新增“清理数据”入口，可勾选清理页面缓存、临时文件、日志、Debug 日志、备份状态、账号和配置；重要数据要求二次确认。
- Docker Compose 中 app 与 AList 服务改为 `restart: unless-stopped`，异常退出或宿主机重启后会自动拉起。

### 安全
- JSON 配置/状态读取失败时不再静默重置默认值，会保留 `.corrupt-*` 备份并报错。
- BBDown Cookie / APP access token 不再通过命令行参数传入，改为每次下载使用临时配置文件，并对输出与 debug 日志脱敏。
- 登录成功后重新生成 session，降低 session fixation 风险。
- `/api` 下非 GET 请求增加同源校验。
- Cookie 导出改为 POST + `EXPORT_COOKIE` 手动确认，并支持 `ALLOW_COOKIE_EXPORT=false` 关闭。
- Session Cookie 支持通过 `COOKIE_SECURE=true` 启用 secure 标记。

### Docker / 发布
- GitHub Actions 支持 `dev` 分支发布 `minori0721/bili-favorites-backup:dev`，`latest` 仍只由 `main` 发布。
- Docker Node 基础镜像更新到 `node:20-bookworm-slim`，并保留 `NODE_IMAGE` 构建参数。
- BBDown 下载支持固定 `BBDOWN_VERSION` 和可选 `BBDOWN_SHA256` 校验。
- Compose 示例改用 `please-change-*` 默认值，并将 AList 示例镜像固定到 `xhofe/alist:v3.41.0`。

### 已知事项
- `npm audit --omit=dev` 仍会报告 `@renmu/bili-api` 传递依赖 `fast-xml-parser` 的中危项；自动修复会降级主依赖，本轮未强行处理。
- 默认 `ADMIN_PASS` / `SESSION_SECRET` 只做文档提示，不做启动拒绝。

## [2.2.2] - 2026-05-12

- 修复收藏夹详情误用本地状态表导致未索引收藏夹显示为空、数量偏少、标题仅显示 BV、UP 显示 Unknown、封面缺失的问题。

### 优化

- 打开详情时会记录当前页视频元数据与收藏顺序，但不会触发下载队列。
- 状态筛选继续基于本地已索引记录，并显示已索引数量提示，避免把局部统计误认为全量结果。

## [2.2.1] - 2026-05-12

### 优化
- 优化任务日志“队列看板”布局，改为固定高度与列内独立滚动，避免大量待下载任务撑乱页面。
- 队列卡片改为两行标题、省略长文本，并用小标签展示重试次数与耗时。
- 队列看板新增基于稳定任务标识的差异更新与移动动画，减少每秒整块重绘带来的闪烁。

## [2.2.0] - 2026-05-12

### 新增
- 任务日志新增“队列看板”，可按“待下载 / 下载中 / 待上传 / 上传中”查看实时任务流转。
- 新增队列快照接口，用于前端稳定展示当前下载、上传和等待队列状态。
- 收藏夹详情记录并使用 B 站收藏夹页码与页内顺序，列表展示更贴近 B 站真实顺序。

### 优化
- 收藏夹详情中的“未上传”不再包含“未上传并失效”，失效视频单独归类统计。
- BBDown 输出文件名过长时会自动截断视频标题并重试，避免长标题视频反复卡在合并失败。
- 调试日志继续保留解析失败线索，便于区分 B 站解析异常和本地文件输出异常。

### 修复
- 修复收藏夹详情接口读取失败状态时未完全绑定收藏夹关系的问题，减少跨收藏夹状态串扰。
- 修复部分超长标题视频因系统文件名限制导致每轮重复失败的问题。

## [2.1.9] - 2026-05-11

### 修复
- 修复 Docker 构建阶段 `apt-get install` 大量 `403 Forbidden` 导致镜像构建失败的问题。
- Dockerfile 调整为：默认使用 Debian 官方源（CI 稳定），国内环境可通过 `ARG APT_MIRROR=tuna` 显式切换清华镜像。

## [2.1.8] - 2026-05-11

### 优化
- 对账提速：AList 目录列表增加短时缓存，同轮多视频复用目录快照，显著减少重复请求。
- 对账限速增强：新增目录维度的节流分配，并保留全局限速与并发控制，降低高并发时对单目录冲击。
- 调度增强：同步运行中再次点击“立即同步/对账”不再直接失败，改为排队到下一轮执行。

### 新增
- 任务日志新增第三档“调试模式”，可与“精简/原始”切换。
- BBDown 在“解析此分P失败”时会自动补跑一次 `--debug --only-show-info` 并保存调试日志到 `data/debug/*.log`。
- 日志持久化：网页日志写入 `data/logs.json`，容器重启后可继续查看最近日志。

### 新增
- 收藏夹详情新增 5 组状态筛选与计数：
  - 全部
  - 已上传
  - 未上传
  - 未上传并失效
  - 已上传且失效
- 新增接口 `GET /api/users/:id/favorites/:mediaId/state-items`，支持按状态过滤并返回统计汇总。

### 修复
- 修复收藏夹详情中“是否已上传”判断未绑定 `mediaId`，导致跨收藏夹状态串扰的问题。
- 修复失败状态按 `userId+bvid` 记录导致跨收藏夹串扰的问题，改为按 `userId+mediaId+bvid` 记录。
- 禁用旧“全局上传证明自动继承为关系已上传”逻辑，避免 AList 实际缺失但界面仍显示已上传。
- 失败任务会在下一轮优先回队列，不再依赖再次翻页命中才可能重试。

## [2.1.7] - 2026-05-11

### 调整
- `解析此分P失败` 不再作为永久失败处理：本轮任务队列不重试，下一轮同步扫描时会重新入队。
- 明确的下架/不可见/`Arg_KeyNotFound` 仍会停止自动重试，避免无意义请求。

## [2.1.6] - 2026-05-11

### 修复
- 修复 BBDown 输出“解析此分P失败”但程序仍显示“下载完成”的误判；现在必须真实产出媒体文件才会进入上传。
- 修复 v2.1.5 关系级状态迁移后，历史已上传视频被当作新视频重新下载的问题；历史全局上传证明会先导入到收藏夹关系，再等待 AList 状态对账确认。
- 修复关系级远端对账回退使用全局 `remoteFiles` 导致跨收藏夹目录误判已上传的问题。
- 下载/上传队列失败与重试现在会进入网页日志；永久解析失败不会在后续扫描中反复自动入队。

## [2.1.5] - 2026-05-10

### 变更
- 备份状态改为按“账号 + 收藏夹 + BV号”记录，同一个视频出现在多个收藏夹时，会分别确认每个 AList 目录是否存在文件。
- 同一个 BV 可复用一次下载结果，并按需上传到多个收藏夹目录，避免重复下载但不再漏传目录。
- 自动初始化扫描加速：初始化阶段每轮补扫更多历史页，手动同步也会扫更深。

### 修复
- 修复跨收藏夹全局去重导致某些收藏夹目录文件数明显少于 B站收藏夹数量的问题。
- 修复远端对账只看目录是否有文件、未按 BV号匹配文件名导致误判“已上传”的问题。
- 修复热页扫描把已完成初始化的收藏夹重新标记为初始化中的问题。

## [2.1.4] - 2026-05-10

### 变更
- 统一上传状态判定为 `videos.backupStatus`，旧 `processedByUser` 不再参与业务判断。
- 手动/自动对账统一走新远端校验路径，历史记录即使没有 `remoteFiles`、只要有 `remotePath` 也可校验。
- 上传完成但缺少必要元数据时，不再误标记为“已上传”，改为回到待补传状态。

- 新增 AList 对账限速参数：`remoteVerifyRateLimitPerSecond`（次/秒）。
- 新增每轮最多补传参数：`remoteRequeueLimitPerCycle`。
- 新增远端校验退避字段：`nextRemoteCheckAt`，用于失败后延迟复查。

### 修复
- 修复“仅按局部数据对账导致误判已上传”的问题。
- 修复远端路径缺失时无法确认文件状态的问题。
- 修复“添加 B站账号”按钮文案乱码。
