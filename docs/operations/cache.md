# 缓存与残片

`temp/`既是下载缓存，也是中断恢复和待补传依据。它不是随时可以清空的临时目录。

## 三类本地内容

| 分类 | 示例 | 自动清理 |
| --- | --- | --- |
| 可恢复 | aria2控制文件、已验证分P、完整本地成品 | 禁止随意清理 |
| 待补传 | 下载完成但AList未最终确认 | 所有目标确认后清理 |
| 待清理残片 | `_invalid`、无法识别旧残片、历史凭据目录 | 仅在页面预览和二次确认后 |

只有有效成品、可继续的aria2控制文件或可确认原始轨道才算“可续传”。失败会话若没有有效分P也没有aria2数据，不会显示为可续传。

## 缓存软上限

默认10GB。达到上限时只暂停启动新下载，上传、远端确认和上传后清理继续运行；释放空间后调度器自动唤醒下载。

## 安全清理

优先使用Web面板“清理数据”查看分类、大小和影响范围。活动会话、可恢复缓存和有效旧成品不会被残片清理自动删除。不要在容器运行时直接执行`rm -rf temp`。
