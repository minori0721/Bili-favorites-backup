# 迁移、备份与回滚

BFB提供轻量迁移包和完整迁移包。两者都使用schema 3清单、逐文件SHA256与原子导入流程。

## 轻量迁移

包含配置、账号、SQLite一致性备份、可读JSON快照、封面和可选日志。适合日常配置迁移或保留状态证明。

## 完整迁移

在轻量包基础上额外包含：

- `temp`下载会话和aria2控制文件。
- 待补传视频。
- 已验证旧分P。
- 历史归档缓存。

完整导出要求没有运行中的传输，并会先显示文件数、原始体积、可续传项和待补传项。完整导入要求目标`temp`为空，非空时返回冲突而不是覆盖。

## 迁移AList归档路径

已有远端归档时，不要直接修改设置页的`alistDest`。请打开“迁移归档路径”，输入新路径并等待扫描预览：新旧路径必须是绝对路径、不能相同或互相嵌套，并且第一段AList挂载名一致。

迁移会遍历整个旧目录树，包括空目录、`_history`和SQLite没有登记的文件。目标同相对路径、同类型、同大小的内容会复用，并在真正切换前再次确认；目标额外内容只展示不修改，大小或类型冲突会阻止开始。实际复制使用WebDAV `COPY`且不覆盖目标，AList异步驱动会在COPY后等待远端可见并确认大小。

复制期间同步、下载、上传、远端确认和画质任务暂停，迁移进度写入SQLite，重启后可继续。全部目标确认后才切换状态、远端记录、任务路径和`config.json`；旧目录不会自动删除。需要清理时先重新核验，再输入`DELETE OLD ARCHIVE`，核验不一致则拒绝删除。

本功能本轮只用模拟WebDAV测试，真实天翼驱动的COPY延迟与目录删除行为仍应在部署阶段验证。扫描、复制、核验、暂停和切换阶段禁止导入或导出迁移包；切换完成进入`cleanup_pending`后可导出备份，但在旧目录处理完成前仍禁止导入。

## 导入安全

文件先流入隔离暂存目录，验证路径、CRC/SHA256、条目数、展开大小、压缩比、配置、账号、日志和SQLite完整性后才切换。导入前自动备份与普通导出使用同一套manifest，并先自检；任一步失败都会保留旧数据。

默认限制：最多100000个条目、100GB压缩包、120GB展开数据、单文件压缩比不超过100。环境变量可调整上限。

::: danger 迁移包是敏感文件
账号数据可能包含B站Cookie和APP token。不要上传到公开网盘、Issue、聊天群或日志收集服务。
:::

## 回滚旧镜像

当前`v2.4.2`开发版的运行数据库使用SQLite schema 5，但迁移包格式仍是schema 3，两者是独立版本。schema 1至4数据库首次升级前会在`data/backups`生成一致性SQLite备份和SHA256摘要；回滚到仍使用schema 4或更早数据库的镜像时，应优先恢复这份升级前备份。

旧版若仍读取`state.json`，应在当前程序停止后导出兼容快照：

```bash
npm run state:export-json -- data/bfb.sqlite data/state.rollback.json
```

检查输出后，再按旧镜像需要放置为`data/state.json`。不要让新旧镜像同时打开同一个SQLite或状态目录。
