# 上传确认

网盘驱动常在WebDAV PUT返回成功后过几秒或几分钟才显示文件。BFB使用两阶段确认，避免把这种延迟误判为上传失败并重复传输。

![上传完成后仍等待远端确认的收藏夹状态](/screenshots/upload-confirming.png){.doc-shot}

## 两阶段状态

1. 上传前检查精确路径。远端已有同名同大小文件时直接记为已确认。
2. PUT返回`201/204`后，记录为`awaiting_verification`，页面显示“已上传·确认中”。
3. 后台立即检查一次，再按约`2秒、10秒、30秒、2分钟、5分钟、10分钟`复查。
4. 大小一致后转为最终确认；所有目标完成后才清理本地文件。

确认期间的404属于“远端可见性延迟”，不会触发上传熔断，也不会重复PUT。

## 超时与冲突

- 10分钟仍不可见：回到`upload_failed`，保留本地文件，约30分钟后再补传；重试前仍会先检查远端。
- 同名同大小：直接确认，不再次上传。
- 同名异大小：不覆盖。当前流程会把可确认的旧版本归档到`_history`后再上传新版；无法安全归档则保留本地并报告冲突。

::: warning 确认中不等于可清理
“已上传·确认中”计入已上传筛选并阻止重复下载，但本地成品仍必须保留，也不能开始画质替换。
:::
