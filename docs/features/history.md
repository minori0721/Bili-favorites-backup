# 分P历史归档

B站视频的分P可能新增、删除、重排或被替换。BFB恢复时重新获取当前详情，并用CID识别分P身份，不依赖标题或文件名。

## 恢复规则

- 相同CID且已验证完整：直接复用。
- 新增CID：只下载缺失分P。
- CID被替换：旧完整分P进入历史，新版重新下载。
- 分P被删除：旧完整分P进入历史。
- 仅顺序变化：按CID重新对应，不重复下载。

远端正式目录保存B站当前版本，旧版保存到：

```text
_history/<快照时间>/
```

历史上传失败不会阻止最新版目标完成，但历史文件会继续以较低优先级补传。最新版与历史文件全部确认前，本地目录不会清理。

## 视频完全下架

如果当前视频已经不可访问，但本地存在经过验证的部分旧内容，系统可归档这部分文件并显示“部分备份”。无法验证的残片只进入清理预览，不会自动上传或删除。

::: info 已归档内容不会被B站更新覆盖
已经上传并确认的历史版本保留在`_history`。B站后续更新只会改变正式目录中的当前版本，不会把历史文件当成待更新目标。
:::
