# 整体运行流程

BFB把一次备份拆成可以独立恢复的阶段。下载成功、上传返回201和最终备份完成是三个不同状态。

<div class="workflow-line">
  <span>扫描收藏夹</span><i>→</i><span>权限判定</span><i>→</i><span>下载与验证</span><i>→</i><span>上传</span><i>→</i><span>远端确认</span><i>→</i><span>安全清理</span>
</div>

## 1. 扫描收藏夹

系统先扫热点页，再逐步补扫历史页。每个视频与“账号 + 收藏夹”的关系分别保存，避免同一个BV出现在多个收藏夹时漏传目标目录。

## 2. 权限判定

下载前先获取视频详情。明确下架转为失效；充电专属且无完整权限转为七日复查；临时风控进入冷却，不会立即循环启动BBDown。

## 3. 下载与本地验证

BBDown固定使用项目fork Release，aria2负责字节级续传。分P以CID识别，媒体文件还需通过大小、时长、流信息与分P覆盖检查，只有完整成品可以上传。

## 4. 上传与远端确认

上传前检查精确远端路径：同名同大小直接确认，同名异大小先归档旧文件。PUT成功后进入“已上传·确认中”，远端同名同大小可见后才最终确认。

## 5. 清理

所有最新版目标和历史文件均已确认后，系统只删除下载会话清单内的本地文件。无法识别的残片不会自动删除，只会进入清理预览。
