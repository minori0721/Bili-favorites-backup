# 异常熔断与恢复

BFB把B站下载接口健康和AList上传健康分开管理。任一侧异常时只暂停会扩大故障的工作，已经拿到下载地址的aria2传输不会被无关熔断中断。

![AList异常时保留本地成品并暂停新下载](/screenshots/upload-degraded.png){.doc-shot}

## AList异常

认证、权限、驱动或持续网络故障会触发上传熔断，并暂停启动新下载，避免本地缓存持续堆积。失败文件保留在`temp`，上传后端恢复后进入持久补传流程。

单个确定性失败会隔离到自己的补传任务，不会永远卡住其他视频。AList返回405时，会根据发生阶段区分临时上传会话异常和真实驱动错误。

![B站播放接口触发风控后的三分钟冷却](/screenshots/download-risk.png){.doc-shot}

## B站风控

网页接口收到`v_voucher`稳定信号后：

1. 保留下载会话和已完成内容。
2. 全局停止启动新BBDown任务3分钟。
3. 冷却结束只放行原失败任务探测。
4. 有APP token时临时用APP接口；没有则继续Web单任务探测。
5. 收到播放地址就恢复健康，不等待整段视频下载结束。

探测再次触发风控会重新冷却固定3分钟，不指数延长。用户选择常驻APP模式时，所有新任务使用APP接口。

## 容器重启

SQLite任务、冷却截止时间和下载会话都持久化。异常退出依靠任务租约过期恢复；正常SIGTERM/SIGINT会停止领取任务、终止BBDown/aria2并等待最多20秒落盘。
