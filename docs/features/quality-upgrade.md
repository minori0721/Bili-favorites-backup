# 共享画质重调

“检查可升级画质”会比较远端归档记录和当前画质设置。只有明确不同的项目默认入选；无法判断的项目单独列出，需要用户确认风险后强制提交。

![同一BV向三个收藏夹目标共享一次新版下载](/screenshots/quality-shared.png){.doc-shot}

## 为什么只下载一次

相同BV、画质、编码、Hi-Res、杜比和文件名模板生成稳定的`artifactKey`。同一成品键的多个收藏夹目标共享一次BBDown下载，再分别执行：

1. 暂存上传。
2. 远端确认。
3. 替换正式文件。
4. 删除旧文件并清理本地成品。

API模式和下载账号不进入成品键，因此切换Web/APP或改派账号不会重复产生同一成品；它们仍进入下载会话指纹，原始轨道不会错误复用。

## 安全边界

- 复用本地成品前重新获取分P CID快照；分P变化时先更新成品。
- 单个目标上传失败不影响其他目标，也不会重新下载共享成品。
- 所有目标完成远端替换和旧文件清理后，才能删除本地成品。
- 旧文件DELETE失败时显示“旧文件清理重试中”，按最长6小时的持久退避继续重试。
- 已上传确认中、充电受限、账号不可用或旧文件资料不完整时，不执行远端替换。
