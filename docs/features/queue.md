# 四列任务队列

队列看板把当前工作按阶段分成“待下载、下载中、待上传、上传中”。SQLite中的全部持久任务不会一次性渲染到页面，内存只领取`queuePrefetchLimit`范围内的少量任务。

![390像素宽移动端上的BFB任务队列](/screenshots/dashboard-mobile.png){.doc-shot}

## 队列卡片怎么看

- **等待/运行/上传时长**是已经经历的时间，不是预计剩余时间。
- **重试倒计时**来自持久任务的`not_before`，容器重启后仍然有效。
- **续传中**表示本地会话存在有效成品、aria2控制文件或可确认原始轨道。
- **已完成 4/13 分P**来自下载会话清单，而不是文件名猜测。
- **下载新版 · 3个目标**表示一次画质下载会向三个远端目标分发。

## 缓存横条

顶部横条汇总本地缓存、可续传会话、待清理残片、充电待检查、上传后端状态和B站下载接口状态。缓存达到软上限时只暂停新下载；上传、确认和上传后清理继续运行。

## 日志视图

除队列看板外，还可以切换：

- 精简模式：用户可读摘要。
- 原始输出：下载和上传过程输出。
- 调试模式：错误、警告和Debug文件线索。

日志通过SSE实时推送，历史日志、实时事件和队列错误使用同一脱敏器。
