# 项目定位

Bili-favorites-backup（BFB）是一个面向**云盘归档**的B站收藏夹持续备份系统。它负责发现收藏夹变化、调用BBDown下载、验证本地成品、通过AList WebDAV上传，并再次确认远端文件可见。

## 适合你，如果

- 你已经使用AList管理阿里云盘、夸克、百度网盘、115等存储。
- 你希望多个B站账号、多个收藏夹分别归档到指定目录。
- 你希望Docker重启后继续下载、补传和远端确认，而不是重新开始。
- 你在意下架、分P替换、充电权限和网盘延迟可见这些长期运行问题。

## 不适合你，如果

- 你主要想搭建Emby、Jellyfin或Infuse媒体库。BFB不负责刮削和播放。
- 你只需要偶尔手动下载单个BV。直接使用BBDown会更简单。
- 你不愿持久化`data`和`temp`，或无法为缓存预留空间。

## BFB保存什么

| 位置 | 内容 | 必须持久化 |
| --- | --- | --- |
| `data/` | SQLite状态库、配置、账号、日志、封面 | 是 |
| `temp/` | 下载会话、aria2断点、待上传成品、历史分P | 是 |
| `alist/` | 内置AList的数据目录 | 使用内置AList时是 |

正式备份始终以恢复时B站当前可访问版本为准。被替换或删除的已验证旧分P会进入远端`_history/<快照时间>/`；无法确认的数据不会被自动删除。

## 关键边界

BFB不会绕过付费、充电、地区或账号权限。充电专属且无完整权限的视频会显示“充电视频”，默认每`7天 ± 12小时`检查全部启用账号；试看不算完整备份。
