# AList 405

HTTP 405表示目标端拒绝当前WebDAV方法。它可能来自AList驱动、反向代理方法限制、路径权限，也可能是上传会话在多文件过程中临时失效。

## 先看发生位置

- 首个文件PUT就405：更像驱动、权限或代理配置错误。
- 同一任务已有文件成功，后续才405：BFB会按临时会话异常在约5、10、30分钟后补传。
- MOVE返回405：影响历史归档、批量重命名和画质替换。
- DELETE返回405：新版和本地成品继续保留，画质清理持久重试。

## 检查清单

1. 用同一AList账号在目标目录手动验证PUT、MOVE、DELETE。
2. 检查反向代理是否只允许GET/POST。
3. 确认BFB填写的是WebDAV可用的AList根地址和正确目标路径。
4. 查看AList日志，区分AList本体与网盘驱动返回。
5. 升级AList前备份数据，并按驱动重新做兼容性验收。

不要把所有405都当成网络超时，也不要通过删除本地文件解除失败状态。
