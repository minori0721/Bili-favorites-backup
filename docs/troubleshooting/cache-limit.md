# 缓存已满

缓存达到软上限后，BFB暂停启动新下载，但上传、远端确认和上传后清理继续运行。待下载任务留在SQLite，不会丢失。

## 处理顺序

1. 检查AList上传是否异常；上传恢复通常会自然释放缓存。
2. 在清理页面区分可续传、待补传和待清理残片。
3. 只删除确认无用的残片，不要清空活动会话。
4. 确认磁盘空间足够后，可在设置中提高`localCacheLimitGB`。

软上限默认10GB，可配置为0至1024GB。它不是磁盘配额：单个已经开始的下载可能使实际使用量略高于上限。

如果宿主机磁盘本身已满，先停止产生新写入并释放其他安全空间；不要直接删除`data/bfb.sqlite-wal`或运行中的`temp`目录。
