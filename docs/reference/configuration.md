# 配置项

设置页保存到`data/config.json`。旧字段会在加载时迁移；表中的默认值来自v2.4.3代码。

## 调度与缓存

| 配置 | 默认值 | 有效范围 / 说明 |
| --- | ---: | --- |
| 收藏夹轮询间隔 | 10分钟 | 最少1分钟 |
| 同一视频分P间隔 | 15秒 | 不小于0 |
| 下载并发 | 1 | 1至5，建议保持1 |
| 上传并发 | 2 | 1至10 |
| AList文件上传间隔 | 10秒 | 0至120秒，只限制真实PUT启动 |
| 本地缓存软上限 | 10GB | 0至1024GB |
| 任务预取上限 | 25 | 5至100 |
| 下载重试次数 | 3 | 0至20 |
| 重试基础延迟 | 5秒 | 1至3600秒 |

`queuePrefetchLimit`只限制内存领取的持久任务数量，不限制SQLite中可以保存的待办总数。

## AList

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| 内部通信地址 | `http://alist:5244` | 必须是HTTP或HTTPS URL |
| 用户名 | `admin` | AList/WebDAV账号 |
| 密码 | 空 | 保存后不在日志输出 |
| 目标目录 | `/bili-backup/videos` | 必须以`/`开头；已有归档时修改请使用“迁移归档路径” |
| 目录布局 | 用户名/收藏夹/视频 | 也可选收藏夹/视频、仅视频 |
| 远端核验并发 | 3 | 1至100 |
| 远端核验速率 | 2次/秒 | 0.5至100 |
| 每轮最多补传 | 20 | 1至1000 |

## BBDown

| 配置 | 默认值 | 可选值 |
| --- | --- | --- |
| 播放接口 | 网页 | 网页、APP |
| 画质 | 自动 | 自动、8K、4K、1080P60、1080P、720P |
| 编码 | 自动 | 自动、HEVC、AVC、AV1 |
| Hi-Res | 关闭 | 开启时必须使用APP接口 |
| 杜比 | 关闭 | 开启时必须使用APP接口 |
| 文件名模板 | `<videoTitle>-<bvid>` | 1至240字符，必须包含`<bvid>` |

APP模式要求所有启用账号都具有扫码登录得到的access token。

## 维护

| 配置 | 默认值 | 有效范围 |
| --- | ---: | --- |
| 旧命名扫描安全上限 | 10000个文件 | 100至100000 |

达到扫描上限会明确显示预览未完成，并禁止执行不完整重命名批次。

## 环境变量

部署安全和日志上限使用环境变量，包括`ADMIN_USER`、`ADMIN_PASS`、`SESSION_SECRET`、`COOKIE_SECURE`、`ALLOW_COOKIE_EXPORT`、`BFB_DEBUG_LOG_RETENTION_DAYS`、`BFB_DEBUG_LOG_MAX_FILES`和`BFB_DEBUG_LOG_MAX_MIB`。参见[安全配置](../operations/security)与[日志](../operations/logs)。
