# 状态含义

BFB的备份状态按“账号 + 收藏夹 + BV号”保存，页面会再汇总同一视频的关系状态。

| 页面状态 | 内部状态 | 含义 |
| --- | --- | --- |
| 待备份 | `discovered` | 已发现，尚未进入下载队列 |
| 排队中 | `queued` | 已有持久下载任务 |
| 下载中 / 续传中 | `downloading` | 正在下载或恢复会话 |
| 待上传 | `downloaded` | 本地成品已验证 |
| 上传中 | `uploading` | 正在向某个目标传输 |
| 待补传 | `upload_failed` | 本地成品保留，等待补传 |
| 已上传·确认中 | `uploaded` | PUT成功，尚未确认远端可见 |
| 已备份 | `verified` | 远端文件最终确认 |
| 部分备份 | `partial_verified` | 下架后仅部分旧内容已验证并归档 |
| 充电视频 | `charging_restricted` | 充电专属且全部当前账号无完整权限 |
| 远端缺失 | `missing` | 备份证明存在，但远端文件核验缺失 |
| 已失效 | `lost` | 视频明确下架或不可访问 |
| 失败 | `failed` | 非上传阶段的失败状态 |

## 什么计入已上传

`uploaded`、`verified`和`partial_verified`都计入“已上传”筛选，但只有`verified/partial_verified`是最终确认状态，也只有它们允许画质重调。

## 什么计入待备份

“充电视频”仍是待备份内容，但不计为普通失败或已失效。权限恢复后会自动回到正常下载流程。

## 视频失效时

- 已确认备份不会因为B站失效而降级或删除。
- “已上传·确认中”的失效视频显示“已下架（确认中）”。
- 有验证过的部分旧内容时可转为“部分备份”。
