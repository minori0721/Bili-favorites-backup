# Dev 测试说明

本文只记录 `dev` 分支相对 `main` 的当前测试内容。确认稳定并合并到 `main` 时，把需要公开给用户的内容整理进 `CHANGELOG.md`，然后重置本文件。

## 当前状态

- 分支：`dev`
- 基准版本：`2.3.1`
- 当前 dev 提交：前端视觉质感、交互状态、站内确认弹窗、移动端细节优化、数据迁移、下架封面缓存、封面跨挂载写入兼容与详情列表长标题布局修复
- 镜像预期：推送 `dev` 分支后发布 `minori0721/bili-favorites-backup:dev`；`latest` 仍只由 `main` 分支发布。
- 本轮定位：在不改变页面大布局、文字颜色和业务交互的前提下，轻量统一 Web 面板质感，并减少前端样式臃肿。

## 当前 dev 变更

- 登录页和主面板增加更柔和的浅色背景层次、半透明面板、轻量边框与阴影，保持原有 Miku 绿主题和文字颜色不变。
- Web 面板抽出玻璃质感相关 CSS 变量，统一面板、卡片、输入框、弹窗、队列、帮助卡片等组件的边框、背景和阴影。
- 将一批纯视觉 `inline style` 改为语义化 class，例如弹窗操作区、表单提示、模板变量区、状态提示、二维码区域和不同宽度的弹窗面板。
- 统一筛选按钮、日志切换按钮、模板标签、帮助按钮等局部控件的圆角、边框和悬停反馈。
- 增加 `backdrop-filter` 不支持时的背景兜底，避免毛玻璃不可用时面板透明度过低。
- 移动端按钮自适应规则改为只作用于账号操作区、设置操作区、弹窗操作区和预览操作区，降低对普通行布局的误伤。
- 统一弹窗打开/关闭逻辑，补充遮罩点击关闭、`Esc` 关闭、关闭后焦点回到触发按钮等键盘可用性细节。
- 新增站内统一确认弹窗，替换删除账号、导出 Cookie、全量扫描并对账、远端重命名、画质重调中的浏览器原生 `confirm()` / `prompt()`。
- Cookie 导出改为站内输入确认流程，必须输入 `EXPORT_COOKIE` 后才允许提交；浏览器剪贴板被拦截时会给出明确提示。
- 保留问号帮助入口，补充更统一的 hover / focus 状态和 `aria-label`，避免只靠 `title` 暴露含义。
- 抽出 `setHidden()`、`setStatus()` 与对应 class，减少前端脚本里直接操作 `style.display` / `style.color` 的临时样式逻辑。
- 统一危险操作按钮、禁用按钮、确认弹窗详情块、确认输入块等交互状态样式。
- 统一收藏夹、视频详情、旧命名文件、画质重调、队列看板等区域的数据兜底显示，避免字段缺失时出现空白或英文占位。
- 统一加载中、空状态、错误提示和 toast 关闭按钮样式，减少散落在脚本里的临时内联样式。
- 优化队列看板在窄屏下的横向浏览体验，保留原四列结构但避免列宽挤压和按钮撑破。
- 扫描到正常 B 站视频信息时，写入标题、UP、封面 URL 的元信息快照；视频失效后不再用“已失效视频”等占位信息覆盖旧快照。
- 新增本地封面缓存：正常封面会异步压缩为半尺寸 WebP q70，保存到 `data/covers/BVxxx.webp`，下架页面和收藏夹详情在在线封面缺失或视频失效时优先使用本地封面。
- 新增「数据迁移」弹窗，可导出 zip 迁移包，包含 `manifest.json`、配置、账号、备份状态、日志、封面缓存和 `indexes/unavailable-videos.json`。
- 新增导入预览与确认导入流程：导入前展示版本、账号数、视频数、关系数、已失效视频数，确认后直接上传 zip 原始包，自动备份当前 `data` 再恢复所选数据。
- 扩展清理数据项目，允许清理封面缓存、导出压缩包和导入前自动备份包。
- 封面缓存落盘时兼容 Docker 跨挂载场景：`temp` 到 `data/covers` 的 `rename` 如果遇到 `EXDEV` / 权限类错误，会自动改为复制后清理临时文件。
- 收藏夹详情和下架清单的视频项增加长标题保护：标题、UP 主和 BV 元信息会省略显示，状态徽章保持固定宽度；窄屏下徽章可换行靠右，避免被挤出列表。

## 测试重点

- 登录页是否正常显示并可进入主页面。
- 账号与同步、全局设置、任务日志三块主区域是否保持原有布局。
- 设置说明、扫码登录、清理数据、收藏夹详情、下架清单、旧命名文件、画质重调等弹窗是否能正常打开、关闭并保持按钮布局。
- 弹窗是否支持遮罩点击关闭、`Esc` 关闭，并且关闭后焦点回到触发按钮。
- 删除账号、导出 Cookie、全量扫描并对账、远端重命名、画质重调是否都走站内确认弹窗，不再出现浏览器原生确认框。
- Cookie 导出是否必须输入 `EXPORT_COOKIE` 后确认按钮才可用，取消时不应调用导出接口。
- 远端重命名和画质重调在预览弹窗内再次确认后，外层弹窗和焦点返回是否保持正常。
- 移动端窄屏下长按钮是否自然换行，不遮挡文字或撑破卡片。
- 队列看板在移动端是否保持四列横向浏览，加载失败时是否显示统一错误空状态。
- 后端字段缺失时，账号名、收藏夹名、UP 主、BV 号、路径、原因等位置是否显示中文兜底。
- 正常视频扫描后是否会在后台生成半尺寸 WebP q70 封面缓存；失效视频列表是否优先使用历史标题、UP 和本地封面兜底。
- Docker 中 `temp` 与 `data/covers` 分属不同挂载时，封面缓存不应再因 `EXDEV: cross-device link not permitted` 失败。
- 收藏夹详情和下架清单里遇到超长标题、超长 UP 主或 BV 号时，右侧状态徽章不应被挤出或裁掉。
- 数据迁移导出是否能生成标准 zip，包内是否包含 `manifest.json`、`data/state.json` 和 `indexes/unavailable-videos.json`。
- 数据迁移导入预览是否只读不写；确认导入是否直接提交 zip 原始包，并先生成 `data/backups/before-import-*.zip`。
- 同步、扫描、对账或下载/上传运行中，导入是否会被拦截。
- 日志控制台仍应保持高对比深色样式，不跟随面板毛玻璃化。
- 构建检查：`npm run build`。
- 静态检查：确认 `src/web.ts` 中没有遗留 `style=`、`style.`、原生 `confirm(`、原生 `prompt(`。

## 建议测试命令

```bash
npm ci
npm run build
npm --prefix . audit --omit=dev
```

> 注意：`npm audit --omit=dev` 可能会报告依赖传递风险；自动修复前需要确认不会降级或破坏主依赖。

## 已知问题 / 暂缓项

- 本轮只做视觉与样式维护性优化，不拆分 `src/web.ts` 的大型内联脚本结构。
- 本地 mock 预览只用于前端视觉与交互检查，必须保持真实 API 外壳 `{ success, data }`，清理数据示例也应按真实后端 7 个清理项模拟；真实数据仍以实际 API 返回为准。

## 合并到 main 时的处理

1. 在 `dev` 环境完成本轮变更对应的测试路径。
2. 把本文件「当前 dev 变更」整理为 `CHANGELOG.md` 的正式版本条目。
3. 确认 README 只保留稳定用户需要看的内容，dev 测试细节继续留在本文件。
4. 合并 `dev` 到 `main` 后，确认 GitHub Actions 只为 `main` 发布 `latest`。
5. 如果要保留下一轮 dev 测试，把本文件重置为新的 dev 状态。
