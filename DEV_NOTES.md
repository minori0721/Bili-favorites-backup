# Dev 测试说明

本文只记录 `dev` 分支相对 `main` 的当前测试内容。确认稳定并合并到 `main` 时，把需要公开给用户的内容整理进 `CHANGELOG.md`，然后重置本文件。

## 当前状态

- 分支：`dev`
- 基准版本：`2.3.1`
- 当前 dev 提交：前端视觉质感、交互状态与移动端细节优化
- 镜像预期：推送 `dev` 分支后发布 `minori0721/bili-favorites-backup:dev`；`latest` 仍只由 `main` 分支发布。
- 本轮定位：在不改变页面大布局、文字颜色和业务交互的前提下，轻量统一 Web 面板质感，并减少前端样式臃肿。

## 当前 dev 变更

- 登录页和主面板增加更柔和的浅色背景层次、半透明面板、轻量边框与阴影，保持原有 Miku 绿主题和文字颜色不变。
- Web 面板抽出玻璃质感相关 CSS 变量，统一面板、卡片、输入框、弹窗、队列、帮助卡片等组件的边框、背景和阴影。
- 将一批纯视觉 `inline style` 改为语义化 class，例如弹窗操作区、表单提示、模板变量区、状态提示、二维码区域和不同宽度的弹窗面板。
- 统一筛选按钮、日志切换按钮、模板标签、帮助按钮等局部控件的圆角、边框和悬停反馈。
- 增加 `backdrop-filter` 不支持时的背景兜底，避免毛玻璃不可用时面板透明度过低。
- 移动端按钮自适应规则改为只作用于账号操作区、设置操作区、弹窗操作区和预览操作区，降低对普通行布局的误伤。
- 统一弹窗打开/关闭逻辑，补充遮罩点击关闭、`Esc` 关闭、关闭后焦点回到触发按钮等键盘可用性细节。
- 统一收藏夹、视频详情、旧命名文件、画质重调、队列看板等区域的数据兜底显示，避免字段缺失时出现空白或英文占位。
- 统一加载中、空状态、错误提示和 toast 关闭按钮样式，减少散落在脚本里的临时内联样式。
- 优化队列看板在窄屏下的横向浏览体验，保留原四列结构但避免列宽挤压和按钮撑破。

## 测试重点

- 登录页是否正常显示并可进入主页面。
- 账号与同步、全局设置、任务日志三块主区域是否保持原有布局。
- 设置说明、扫码登录、清理数据、收藏夹详情、下架清单、旧命名文件、画质重调等弹窗是否能正常打开、关闭并保持按钮布局。
- 弹窗是否支持遮罩点击关闭、`Esc` 关闭，并且关闭后焦点回到触发按钮。
- 移动端窄屏下长按钮是否自然换行，不遮挡文字或撑破卡片。
- 队列看板在移动端是否保持四列横向浏览，加载失败时是否显示统一错误空状态。
- 后端字段缺失时，账号名、收藏夹名、UP 主、BV 号、路径、原因等位置是否显示中文兜底。
- 日志控制台仍应保持高对比深色样式，不跟随面板毛玻璃化。
- 构建检查：`npm run build`。

## 建议测试命令

```bash
npm ci
npm run build
npm --prefix . audit --omit=dev
```

> 注意：`npm audit --omit=dev` 可能会报告依赖传递风险；自动修复前需要确认不会降级或破坏主依赖。

## 已知问题 / 暂缓项

- 本轮只做视觉与样式维护性优化，不拆分 `src/web.ts` 的大型内联脚本结构。
- 本地 mock 预览只用于前端视觉检查，部分测试账号字段可能不完整；真实后端数据以实际 API 返回为准。

## 合并到 main 时的处理

1. 在 `dev` 环境完成本轮变更对应的测试路径。
2. 把本文件「当前 dev 变更」整理为 `CHANGELOG.md` 的正式版本条目。
3. 确认 README 只保留稳定用户需要看的内容，dev 测试细节继续留在本文件。
4. 合并 `dev` 到 `main` 后，确认 GitHub Actions 只为 `main` 发布 `latest`。
5. 如果要保留下一轮 dev 测试，把本文件重置为新的 dev 状态。
